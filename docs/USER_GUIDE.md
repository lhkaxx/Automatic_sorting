# 用户指南 — 考场自动排座系统

## 快速开始

### 1. 启动应用

```bash
cd Automatic_sorting
env_kaoshi\python.exe app.py
```

打开浏览器访问 `http://localhost:5000`

### 2. 上传数据

进入 **📤 上传数据** 页面，上传 5 个 CSV 文件：

- **students.csv**（必需）— 学生花名册
- **classes.csv**（必需）— 班级信息
- **classrooms.csv**（必需）— 考场信息（含座位布局）
- **subjects.csv**（必需）— 考试科目
- **teachers.csv**（可选）— 教师信息（监考分配用）

#### CSV 格式要求

- 编码：UTF-8（自动检测 BOM）
- 列头必须与模板一致（见 data/ 目录下的示例文件）
- `seat_layout` 格式：`{rows}×{cols}` 或 `{rows}×{cols}!(row,col);(row,col)`

#### 网格编辑器

如果不想准备 classrooms.csv，可以使用页面上的**网格编辑器**手动添加考场：

1. 填写教室编号、楼号、楼层、容量
2. 设置行数和列数
3. 点击「🔄 渲染网格」
4. 点击格子切换可用/阻塞状态
5. 点击「➕ 添加考场」

### 3. 配置约束

进入 **⚙️ 配置** 页面：

1. **考试场次** — 勾选要排座的科目，设置考试时间
2. **班级隔离** — 选择隔离模式（推荐：四邻无同班）
3. **考场管理** — 保持默认「考场容量上限」开启
4. **学生分配** — 可选：作弊/扣分考生前排
5. **教师监考** — 如果上传了教师数据，可选回避任教班级
6. **算法选择** — 默认 SA（推荐），可选 ACO 对比
7. **参数调整** — 默认参数适用于大多数场景

点击「🚀 开始排座」触发计算。

### 4. 查看结果

排座完成后自动跳转到**考场视图**。4 个标签页：

#### 🏫 考场视图
- 每个考场显示为座位网格表
- 不同班级用不同颜色标识
- 空座显示 "—"，阻塞格显示深灰
- 悬停座位看详细信息（学号、姓名、班级、作弊/扣分标记）

#### 👤 考生视图
- 搜索学号或姓名实时过滤
- 每个学生一张卡片，显示所有科目的座位
- 作弊 ⚠ 和扣分标记清晰可见

#### 👨‍🏫 教师视图
- 表格展示每位监考教师负责的考场
- 如果回避任教班级约束启用，违规项标红 ⚠

#### 📊 班级视图
- 柱状图：各班在各考场的分布人数
- 下拉框选择班级 → 高亮该班所有座位
- 查看每个考场内该班学生的座位散布

### 5. 评估报告

在结果页面点击「📊 生成评估报告」：

- **约束满足率** — 绿色 ≥ 95%
- **班级隔离度** — 越接近 100% 越好
- **收敛曲线** — SA/ACO 迭代过程的 energy 变化
- **考场利用率** — 每个考场的已用/可用比例
- **违规详情** — 硬违规（红）和软建议（黄）

### 6. 导出

- **📥 JSON** — 完整结构化结果，含学生信息
- **📥 CSV** — 扁平化名单，便于打印和导入其他系统

## 常见问题

### Q: 上传 CSV 后没有数据？
A: 检查 CSV 编码（UTF-8）、列头名称是否与模板一致。空文件只有表头不会报错。

### Q: 排座失败"可用座位少于考生人数"？
A: 考场容量不足。检查 `classrooms.csv` 的 `seat_layout` 是否正确，或手动添加更多考场。

### Q: 算法运行很慢？
A: 调整参数：减少重启次数、提高冷却速率、降低初始温度。对于 < 50 人的规模，几秒即可完成。

### Q: 有作弊记录的学生没有被安排到前排？
A: 确保在「配置」页面的「学生分配」中勾选了「作弊考生前排」。

### Q: 班级隔离后仍有同班相邻？
A: 检查隔离模式是否正确勾选。如果考生人数多、考场少，可能存在无法完全隔离的情况，此时会显示在违规详情中。
