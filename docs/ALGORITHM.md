# 算法文档 — 考场自动排座系统

## 模拟退火 (Simulated Annealing, SA)

### 核心思想

模拟退火模拟物理退火过程：高温时粒子剧烈运动探索全局，低温时趋于稳定收敛到最优解。在排座问题中，"温度"控制接受劣解的概率。

### 算法流程

```
1. 初始化：随机分配所有学生到可用座位
2. 计算当前 energy
3. 在降温循环中：
   a. 邻域扰动 → 生成新方案
   b. 计算 delta = energy(new) - energy(current)
   c. 如果 delta < 0：接受新方案
      否则：以 exp(-delta / T) 概率接受
   d. 如果新方案更优，更新全局最优
4. T *= cooling_rate（降温）
5. 重复 3-4 直到 T < T_end 或超时
6. 多次重启（restart_count），取各次最优
```

### 目标函数 (Energy)

```
energy = 硬约束违规数 × 1000 + 空座位数 × 1 + 均衡惩罚 × 10
```

- 硬约束违规权重最高，一旦有违规 energy 急剧升高
- 空座位数确保座位利用率最大化
- 均衡惩罚是软约束，各考场人数标准差越大惩罚越高

### 邻域扰动策略

| 概率 | 操作 | 描述 |
|------|------|------|
| 90% | 交换 | 同一考场内交换两个已占座位的学生 |
| 8% | 移动 | 将一个学生从已占座位移到空座 |
| 2% | 跨考场互换 | 不同考场的两个学生互换座位 |

### Metropolis 接受准则

```
P(接受劣解) = exp(-delta / T)
```

- 温度 T 高时 → 高概率接受劣解 → 探索
- 温度 T 低时 → 低概率接受劣解 → 收敛

### 参数说明

| 参数 | 默认值 | 建议范围 | 含义 |
|------|--------|----------|------|
| T_start | 100.0 | 50-500 | 初始温度，越高探索越充分 |
| T_end | 0.01 | 0.001-0.1 | 终止温度，越低收敛越彻底 |
| cooling_rate | 0.9995 | 0.99-0.9999 | 冷却速率，越接近 1 降温越慢 |
| iterations_per_temp | 100 | 50-500 | 每个温度下的迭代次数 |
| restart_count | 3 | 1-10 | 退火重启次数 |

### 收敛性

- 理论上：无限慢降温可保证全局最优
- 实践中：针对 ~200 学生规模的排座，默认参数通常能在 10-30 秒内收敛到可行解

## 蚁群算法 (Ant Colony Optimization, ACO)

### 核心思想

模拟蚂蚁觅食行为：蚂蚁在路径上留下信息素，信息素浓度高的路径被后续蚂蚁优先选择。在排座问题中，"路径"是"学生→座位"的分配选择。

### 算法流程

```
1. 初始化信息素矩阵 tau[N_students × N_seats] = 1 / N_seats（均匀分布）
2. 每轮迭代：
   a. N_ants 只蚂蚁各自构建排座方案
      - 对每个学生，按概率分布选择座位：
        P(student, seat) ∝ tau(st, seat)^α × eta(st, seat)^β
   b. 评估每只蚂蚁的方案 energy
   c. 信息素蒸发：tau *= (1 - ρ)
   d. 精英蚂蚁强化：对最优蚂蚁的方案增加信息素
      delta_tau = 1 / energy(best_solution)
3. 重复直到 n_iterations 或超时
```

### 参数说明

| 参数 | 默认值 | 含义 |
|------|--------|------|
| n_ants | 20 | 每轮蚂蚁数量 |
| n_iterations | 200 | 迭代轮数 |
| evaporation_rate ρ | 0.1 | 信息素蒸发率，防止过早收敛 |
| α (信息素权重) | 1.0 | 越大越依赖历史经验 |
| β (启发式权重) | 2.0 | 越大越依赖局部启发信息 |

### SA vs ACO 对比

| 维度 | SA | ACO |
|------|-----|-----|
| 收敛速度 | 快 | 较慢 |
| 解的质量 | 好 | 可能更好（更多样化） |
| 参数敏感性 | 中等 | 较高 |
| 推荐场景 | 默认首选 | 对比验证 |
